# SkillTracker Frontend 

Пользовательский интерфейс (Frontend) сервиса SkillTracker, представляющий собой одностраничное приложение (SPA).

## 🛠 Технологический стек
- **Фреймворк**: React 18
- **Сборщик**: Vite 6
- **Язык**: TypeScript 5
- **Маршрутизация**: React Router v6
- **Управление состоянием и кэширование**: React Query (TanStack Query v5)
- **Стилизация**: TailwindCSS 3
- **Компонентная база UI**: Radix UI (основа для Shadcn UI)
- **Анимации**: Framer Motion
- **Работа с формами**: React Hook Form + Zod (валидация схем)
- **Иконки**: Lucide React
- **Графики и дашборды**: Recharts
- **HTTP клиент**: Axios
- **Утилиты для дат**: date-fns

## 🚀 Локальный запуск (режим разработчика)

Убедитесь, что у вас установлен **Node.js**.

1. Перейдите в папку фронтенда:
   ```bash
   cd FrontendSkillTracker
   ```
2. Установите все npm-зависимости (включая dev-зависимости):
   ```bash
   npm install
   ```
3. Запустите dev-сервер:
   ```bash
   npm run dev
   ```

Приложение будет доступно по умолчанию по адресу `http://localhost:5173` (или по порту, выделенному Vite).

## 📦 Сборка для Production (Docker)

Для продакшен-сборки проекта используется скрипт:
```bash
npm run build
```
Команда предварительно прогоняет статическую типизацию (`tsc -b`) и использует Vite для сборки минифицированных JS/CSS бандлов.

**Развертывание через nginx:**
Проект содержит `Dockerfile` и настроенный `nginx.conf`. Сборка образа проходит в 2 этапа (multi-stage build):
1. Сборка статики через окружение node.
2. Проксирование и хостинг статических файлов с помощью легковесного сервера `nginx:alpine`.
В общей сети docker-compose фронтенд слушает и отдаёт файлы на порту `80` (который проброшен для вас на порт `3000`).

## 📁 Структура исходного кода (`src/`)

```
src/
├── components/   # Переиспользуемые UI-компоненты (на основе Radix UI/Shadcn), кнопки, диалоги и т.д.
├── pages/        # Модули экранов/страниц маршрутизатора (Dashboard, Tasks, Login)
├── lib/          # Утилиты, настройки axios-инстанса, utils (например, tailwind-merge)
├── App.tsx       # Корневой компонент с настройкой роутера
└── main.tsx      # Точка входа в React-приложение (рендеринг в DOM)
```
